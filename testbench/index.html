第 1 步 · 对话大模型

MemoWeft 跟你对话、整理记忆都靠它。这一步三项都必填。任何 OpenAI 兼容的 /v1 端点都行（云端如小米 MiMo，或本地 Ollama / LM Studio）。

模型服务的网址，一般以 /v1 结尾。就是你在模型厂商后台拿到的那个「API 地址」。

末尾别多写 /chat/completions，只到 /v1 就行——代码会自己补后面那段。

接口地址不能为空

模型厂商给你的口令，通常以 sk- 开头。填错会提示「LLM 请求失败」。

这是私密口令，别截图发人、别提交进 git。生成的 .env 一定要加进 .gitignore。

密钥不能为空

要用哪个模型，照厂商文档里的「模型 ID」原样填。

名字要跟厂商文档一字不差（含大小写和中划线），填错会报「模型不存在」。

模型名不能为空

第 2 步 · 写路径小模型可选 · 可跳过

整理经历片段、更新对你的了解、猜测原因这些「后台重活」如果也用大模型会慢。配一个本地小快模型专门干这些，能明显提速。不配也行——写路径会自动回退到第 1 步的对话大模型，行为和以前一样、不会崩。

🛠 想用本地小模型？先做这几步（照抄即可）

  1. 装 Ollama（本地跑模型的工具），到官网 ollama.com 下载安装，装完命令行能敲 ollama 就算好了。
  2. 在 11435 端口起 Ollama 服务（作者环境用这个口，避开 codex 占用的 11434）： OLLAMA_HOST=127.0.0.1:11435 ollama serve 已经在跑就不用重起（和下面向量步骤是同一个 Ollama）。
  3. 拉一个小模型（作者验证用 qwen2.5:3b，约 2G）： OLLAMA_HOST=127.0.0.1:11435 ollama pull qwen2.5:3b 显存只有 6G、和向量模型一起爆显存的话 → 降到 qwen2.5:1.5b。
  4. 验证通了：拉完不报错、ollama list 能看到 qwen2.5:3b 就 OK。下面三格照着填。

本地 Ollama 的地址，端口和上面起服务用的一致（这里是 11435）。

本地 Ollama 不校验密钥，随便填个占位就行，一般写 ollama。

上一步 ollama pull 拉的那个模型名，原样填。

这三格要么都填、要么都留空。只填一两格会拼出半截配置——留空整组会自动省略。

第 3 步 · 向量模型（嵌入）可选 · 可跳过

向量模型让 MemoWeft 能「按意思找」你说过的话（比如问「喝什么」能翻出你说过「喜欢喝茶」）。不配也能跑——语义召回降级为空，「对你的了解」照样在攒，只是回话时不会主动带上你的偏好。

🛠 想开启语义召回？先做这几步（照抄即可）

  1. 装 Ollama（同上，官网 ollama.com 下载）。如果第 2 步已经装好了，跳过这条。
  2. 在 11435 端口起 Ollama 服务（和第 2 步同一个 Ollama，起过就不用再起）： OLLAMA_HOST=127.0.0.1:11435 ollama serve
  3. 拉嵌入模型 bge-m3（多语言、中文好，1024 维，约 1.2G）： OLLAMA_HOST=127.0.0.1:11435 ollama pull bge-m3
  4. 验证通了：ollama list 能看到 bge-m3 就 OK。下面三格照着填。

本地 Ollama 的地址，端口和起服务用的一致（11435）。和写路径小模型是同一个 Ollama。

本地 Ollama 不校验，填占位 ollama 即可。

上一步拉的嵌入模型名，原样填 bge-m3。

这三格要么都填、要么都留空。留空整组会自动省略，召回降级为空、「对你的了解」照写。

第 4 步 · 部署选项

最后一项：这套 MemoWeft 是否带体验界面。这决定生成的 .env 里 MEMOWEFT_EXPERIENCE_UI 是 on 还是 off。

MemoWeft · 会慢慢记住你的那一层

你像平时一样聊天、随口提一点自己，它就在旁边顺手一句句记下来、慢慢把这些拼成「它对你的了解」。不用填表、不用打标签——聊着聊着，它就更接得上你说过的话。

📦 备份 / 迁移 · 便携记忆包
把 MemoWeft 对你的全部记忆（原话 / 经历片段 / 理解条目 + 溯源链）打成一个 .bundle.json 文件，可备份、可搬到另一台机器导入。
向量检索索引不进包（那是派生物）——导入后点一次「更新画像」即可重建召回。
⚠ 危险操作
只把上面的参数旋钮还原成出厂值，不删任何数据；大多数改完即时生效，个别项需新会话（见对应提示）。
⚠ 恢复出厂 · 清空全部数据
永久删除 MemoWeft 攒下的全部数据：你说过的原话（记忆线索 evidence）、整理出的经历片段（event）、以及它对你形成的理解（理解条目 cognition），连同检索索引一起清空。
和上面的「恢复默认参数」互不相干——那个只还原旋钮、不碰数据；这个只清数据、不动参数。不可恢复。
