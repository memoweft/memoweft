# AGENTS.md · MemoWeft

极简开工说明。接手先读三份：**本文件、`README.md`、`CURRENT.md`**。读完就够动手了。

## 这是什么

MemoWeft 是一个**库**（npm 包 `memoweft`），不是应用。它把"对一个人的理解"存成可追溯、可迁移、区分事实与猜测的记忆资产，交给宿主 `import`。它不聊天、不做人设、不画界面——那是宿主的事。

三层边界：

- **Core**（`src/`）——记忆怎么正确地存在：三层数据模型、置信度、召回、公共 API。库的本体。
- **Host**（`apps/memoweft-host`）——怎么使用和管理记忆：聊天、界面、多会话、备份。参考实现，不是产品。
- **Plugin**（`plugins/`）——扩展能力：采集器、体验人设。插件只请求，Host 审核，Core 执行记忆规则。

## 默认只读这些

- `README.md` —— 项目是什么、怎么用。
- `CURRENT.md` —— 现在在做什么、能做什么、不能做什么、怎么算完成。
- 与当前任务**直接相关**的源码文件。

**不要**默认通读 `docs/internal/`、项目地图、历史路线、`logs/`。它们是背景资料，需要查某个旧决策时再翻，别当开工必读。

## 规矩（就这几条）

- **不扩大任务范围。** 只做 `CURRENT.md` 里写的那件事；顺手发现的别顺手做，记下来问。
- **改公开 API、数据库 schema、权限 / 授权模型前，先说清影响面**（谁在用、破坏性在哪、要不要留 deprecated 别名），确认了再动。
- **认知纪律是核心，不许"顺手优化"掉**：记≠信、冲突先暴露不自动合并、置信度库自算不听 LLM 自报、分型过期。拿不准算不算核心，就当它是，先问。
- **改完必须三绿**：`npm run typecheck && npm test && npm run build` 全过才算完成。

分支 / 提交 / 依赖最小化等细则见 `CONTRIBUTING.md`——用到再翻。
