# MemoWeft 档2 起步 · 上本地模型（第①块 · 验证）· 给 Coder

> **设计者**：产品经理（2026-07-01）。
> **定位**：档2 那"缠在一起的一坨"（本地模型 / 按授权路由 / 真采集器 / distill 雷 / §6 折中）里，**"上本地模型"是钥匙、第一步**。
> **但这一步只做"验证"**：所有者当前开发机显卡仅 **6G**，先装**小模型**、让写路径走本地，**验证"本地到底能不能顶写路径"**（质量 + 速度），拿真实数据再决定下一步（要不要上"按授权路由"、要不要换更大的模型）。
> **纪律**（AGENTS.md）：碰核心先出方案；改完 docs-sync。

## 1. 做什么（装 + 配 + 验证）

### 1.1 装本地小模型
```
OLLAMA_HOST=127.0.0.1:11435 ollama serve   # 若没在跑先起（和嵌入 bge-m3 同一个 ollama）
OLLAMA_HOST=127.0.0.1:11435 ollama pull qwen2.5:3b
```

### 1.2 配成「写路径模型」
`.env` 加三行（治慢做的 `llmPool` 已支持 `DLA_WRITE_LLM_*`，缺则回退对话大模型——**配上就生效，大概率不用改代码**）：
```
DLA_WRITE_LLM_BASE_URL=http://localhost:11435/v1
DLA_WRITE_LLM_MODEL=qwen2.5:3b
DLA_WRITE_LLM_API_KEY=ollama          # ollama 不校验，随便填
```
→ 写路径（distill / consolidate / attribute / trends）走本地 qwen；对话仍走云端大模型（`llmPool.for('chat')`）。

### 1.3 验证 + 报告（给所有者主观验收）
- 起测试台，跑验收场景：注入两三条"凌晨 2–3 点玩游戏"observed + 聊"没睡好" + 触发更新画像。
- 看本地 qwen 跑出来的：
  - **速度**：`logs/run-*.jsonl` 里 `profile_update` 记录的 `timings`（各步 ms），**对比云端 mimo 的 ~30s/步**。
  - **质量**：画像 / 归因内容合不合理（"凌晨游戏"和"累"联得上吗？假设像话吗？）。
  - **结构跟随**：有没有 JSON 解析失败 / 跟不上格式（小模型常见毛病，`consolidate`/`attribute` 输出 JSON）。
- 产出一份**对比小结**：本地 3b vs 云端 mimo —— **快多少、质量掉多少、结构跟不跟得上、能不能顶**。

## 2. 不做（Out）

- ❌ **不做**"按 `allowCloudRead` 路由本地/云端"——那是**下一步**，等这轮验证了本地质量再上（在 `llmPool` 上加 tier 维度）。
- ❌ 不做真采集器、不碰 distill 雷、不碰 §6 折中（等"按授权路由"那步才碰）。
- ❌ 不动认知层算法 / 不改归因逻辑。

## 3. 注意

- **6G 显存**：qwen2.5:3b（~2G）+ bge-m3（~1.2G），ollama 按需加载够跑；**若爆显存 → 降 `qwen2.5:1.5b`**（如实记一笔）。
- **本地质量大概率不如云端**（尤其 consolidate/attribute 要"判断 + 跟结构"）——**这正是要验的，别预设乐观、别粉饰**。跟不上结构（JSON 坏、漏字段）就如实记下来，这是"3b 顶不顶得住"的关键证据。
- 嵌入 bge-m3 和 qwen 同一个 ollama（11435），互不冲突。

## 4. 影响面

| 动作 | 位置 |
|---|---|
| 装模型 | `ollama pull`（本地，不进仓库） |
| 配置 | `.env`（加 `DLA_WRITE_LLM_*`） |
| 代码 | **大概率不改**（`llmPool` 已支持）；若读 .env 有问题查 `src/llm/pool.ts` |

## 5. 产出 / 验收

- [ ] `qwen2.5:3b` 装好、写路径走本地**跑通不报错**。
- [ ] 一轮验收场景的**对比数据**（本地 vs 云端：速度 / 质量 / 结构跟随）。
- [ ] **结论**：本地 3b 能不能顶写路径 / 要不要换更大的 → **交所有者定下一步**（质量是主观验收，你给数据、别自己拍板）。
- [ ] 若动了代码：`typecheck` / `test` / `build` 绿；`docs-sync`（STATE/LOG）。

## 6. 协作纪律

1. 这步**只"装 + 配 + 验证"**，别顺手上路由 / 采集器 / 碰 distill 雷。
2. `.env`、装模型你来；**"质量够不够顶"的主观判断交所有者**——你负责把客观数据（速度 / 报错 / JSON 跟不跟得上）摆清楚。
3. 要改的代码超出"配 .env"预期、或方案与代码冲突 → **停下说一声**。
