# MemoWeft 后续步骤大纲（11 步 · 商用线 + 功能线合排）

> 这是全项目的后续步骤总排序（2026-07-04 定，作者过目版）。每一步真正开工前，由架构会话按**当时的代码现状**细化成施工任务书并独立校对（`tasks/0.3.0/` 那套就是样板），这里不写行号、不写规则——规则在 `tasks/0.3.0/README.md` 和 AGENTS.md 里已有。

## 十一步

**1 · 0.3.0 补漏加固**（任务书已在 `tasks/0.3.0/`，随时开工）
隐私 B 下沉、发布保险丝、JSON 解析统一、busy_timeout、扫尾、SQLite 驱动抽缝（Node 20/22 可用）、全开源声明。
完成标志：隐私三红线全绿；Node 22 装 better-sqlite3 能跑全测试。

**2 · 接口契约（Memory Surface Contract v1）**
公共 API（`createMemoWeftCore`、`recall / ingest* / memory.*` 的入参与返回）定成带稳定性分级（stable / experimental）的契约文档+类型；破坏性变更政策成文。它是第 7 步插件契约和第 10 步 API 收口的共同地基。顺手同步公开 ROADMAP 的过期口径（0.3.0 节标题、lint 写死 ESLint/Prettier）。

**3 · 质量证据**
eval 约 20 个对话用例（断言纪律：冲突暴露不合并、情绪封顶、记≠信）+ README 对比表 + 覆盖率徽章 + lint 关卡 + SECURITY.md + issue/PR 模板 + 维护声明（单人+AI 维护、响应预期，诚实写明）+ 性能实测（灌 1 万条出 updateProfile 耗时/召回延迟数字）。发布走 CI+provenance 视作者是否托管 npm token。
完成标志：评估者的"凭什么信"和"能撑多少"在文档里有答案。

**4 · 英文化与模型兼容（0.4.0）**
提示词/模板问法/兜底文案/报错抽成中英双语层（默认语言作者定）；INSTALL、integration 英文版；examples 扩到 3 个、以包名为入口；temperature 可配、响应解析兼容 reasoning 分段；hostId 默认名在此拍板改掉。随车半步：文档明示"当前明文落盘，磁盘加密属宿主/系统责任"。
完成标志：英文开发者从 npm 首页到跑通全程零中文、零求助。

**5 · 图谱前端 G2**（框架闭环欠账）
后端 payload 已有（G1 ✅），接 `/api/memory-graph` + 力导向图，记忆管理可视化收尾。工作量小、演示价值大。

**6 · 本地模型档2（hybrid 路由）**（框架闭环欠账，功能线最重要一步）
observed 不上云的证据交本地模型消化成画像——现状这类证据永远挂着不被消化，"行为观察"卖点只有手动授权上云才真跑通。`filterCloudReadable` 的"假设写模型=云端"前提要升级为按当前模型是云/本地决定筛不筛。做完这步采集线才算真闭环。

**7 · 插件契约 v2（10-A）+ 采集器跨平台**
hooks / PluginContext 从"预留"转正式（踩在第 2 步契约上）；采集器现只支持 Windows，补 macOS/Linux 或明说不做。

**8 · 生态获客（0.5.0）**
MCP 服务器（做完**提交 registry 收录**，本地能跑不算完）；一个适配器（Vercel AI SDK 或 LangChain JS 二选一，作者定）；LLM token 用量观测（宿主能算钱）。

**9 · 星瑶最小宿主（9-A，作者个人产品线，可选）**
独立的星瑶体验宿主。与"宿主只是演示"的拍板有张力——做不做、做多深由作者定；库侧只保证第 2/7 步把接口与插件契约铺好。

**10 · 收口发 1.0**
API 出口收窄（现状 51 值 + 120 类型全裸奔，按第 2 步分级收）；正门补全（proposeAsk / revisitConflicts / expire / aggregateTrends 进门面）；画像膨胀治理（consolidate 全量塞 prompt）；多 subject 收口（resetSubject 误清全部向量等）；会话实例缓存上限；投毒防护（深度作者定）；加密落盘做或明示；灾备（WAL+备份策略一起改，0.3.0 T4 欠的）；删除影子级联说明；第三方个人数据合规明示（采集会带到旁人信息，责任划给宿主）；semver 承诺生效 → 五关验收 → 发 1.0。

**11 · 召回质量 v2**（可与第 10 步并行）
相似度门槛启用（minSimilarity 现默认 0=关）、purpose/scope 过滤、召回解释、用户负反馈。

## 贯穿项（不占步骤）

从第 4 步起持续攒**采用证据**：npm 周下载基线、第一个非作者的真实接入、公开案例。1.0 的采用关靠它。

## 顺序逻辑（三句话）

先堵漏（1），再立契约（2）——后面一半的步骤踩在契约上；证据和英文化（3-4）打开信任与触达；功能线（5-7）还清框架闭环欠账，生态（8）拿闭环去获客；最后收口发 1.0（10）。
