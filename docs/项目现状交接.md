## 1. 这是什么项目（定位 + 目标）

**MemoWeft = 套在大模型 / Agent 外部的"用户认知与上下文框架"。** 它持续接收用户授权的对话与行为证据，把对一个人的理解沉淀成**独立于任何模型、可追溯、可演化、可迁移**的认知资产；在需要时提供带置信度和边界的用户上下文。

- **形态**：插件 / 库（npm 包），被宿主（Hermes、星瑶…）`import`，随宿主进程生死。**MemoWeft 不做聊天 / 角色 / 语音**——那是宿主的事。
- **一句话差异**：别人的记忆框架管"存储和检索"；MemoWeft 在记忆之上多一层 **"认知纪律"**（信什么、多有把握、哪些不能越界、能不能接受纠正）。
- **由来**：作者想做桌面助手"星瑶"+ 微调"另一个我"，撞墙（模型刻板、数据混乱、记不住）。想通了：**对一个人的理解不能锁死在模型权重里，必须活在模型之外、可带走、可更新。** 所以"跨模型可迁移"是立身之本。

**验收标准是主观的**：项目所有者用着觉得"它真的在理解我"，不是 benchmark 跑分。
**"越用越懂"分两层**：① 事实 / 偏好层（说过做过的）→ 大概率能做到；② 深层假设层（性格、没说出口的动机）→ 是赌注，靠"低置信 + 证据校准 + 对抗确认偏误"兜底，不追求一次猜准。
**核心规格**："画像可以不准、可以有偏差，但不能偏离太大"——不靠算得准，靠**它知道自己可能错、并有机制去纠**。

## 2. 核心架构（三句话）

- **读写解耦 + 一份真相**：读路径同步轻快（服务当前对话），写路径批量可延后（长期积累），不是一条流水线。
  - **读路径**：用户消息 → 工作记忆窗口（最近几轮）→ 召回相关认知 → 注入回话。默认仅 1 次模型调用。
  - **写路径**：原话 → **证据层**（记≠信，先存原料）→ **事件化 distill**（近期对话总结成带情境的事件）→ **画像 consolidate**（LLM 提认知 → MemoWeft 自算把握度 → 溯源回原话）。
- **三层数据**：`evidence` 原料 → `event` 事件 → `cognition` 判断。存储用 SQLite，向量召回用本地嵌入。
- **认知层规则（MemoWeft 的魂，差异化全在这）**：记≠信 / 来源强度分层（亲口>推断>观测）/ 一条认知多维属性 / **禁止系统自证**（MemoWeft 自己的话、用户沉默都不算证据）/ **冲突先暴露不自动消解** / 可解释假设低置信 / 防过度自信 / **分型过期**（情绪快忘、偏好不忘）/ 只说"收下到什么程度"不说"证明为真" / 用户可随时查改删。
- **宿主边界**：MemoWeft 只给理解（事实、把握度、边界、冲突、沟通偏好）；宿主定表达（语气、角色、是否开口）。

## 4. 已完成的全部内容（按阶段）

### 阶段 0 · 地基 ✅
证据层（`evidence` 13 字段：来源强度 / 双时态 occurred+recorded / 三种授权位 / 幂等 / 纠正指向）+ 对话感知 `perceive` + 回话编排 `Conversation`（带最近几轮窗口）+ 可观测日志 + 测试台骨架。助手回话不落证据（禁止系统自证）。

### 阶段 1 · 画像 + 召回 ✅
- 多维 `cognition`（content_type / formed_by / confidence / cred_status / scope / 有效时间窗 + 溯源链）。
- **把握度 MemoWeft 自算**（按来源强度起步分 + 支持加分 - 反对扣分），**不采信 LLM 自报**。
- 事件化 `distill`（近期对话 → 带情境事件，只总结用户话、禁止自证）。
- 向量召回 `VectorRetriever`（SQLite 存向量 + JS 余弦）+ 本地嵌入器；召回相关认知注入回话。

### 阶段 2 · 纠正闭环 ✅
`consolidate` 改成**增量更新**：未消化事件 + 现有画像 → LLM 判 `new / reinforce / correct / conflict`。
- `correct`（用户明确纠正）→ 旧的标失效保留（invalidAt）+ 采纳新的。
- `conflict`（模糊矛盾）→ 标 `conflicted` 暴露、两条都留、不自动选边。← 这是 MemoWeft 区别于 Graphiti"自动作废旧事实"的分水岭。

### 阶段 3 · 归因 + 带证据主动询问 ✅（真模型主干跑通）
- **M4 归因** `attribute`：现象（state 认知）+ 时间窗证据 → LLM 推可解释假设（`contentType='hypothesis'`、低置信封顶、挂证据、可推翻）。一次只归因最近一条现象、禁"用一个抱怨解释另一个抱怨"、支撑 ≤2 原因 + 1 锚点。
- **M5 带证据主动询问** `proposeAsk`：挑低置信假设（保守时机：候选/低置信、把握度"将信将疑"、一轮 1 个、问过不再问）→ 产出"该问什么 + 附什么证据 + 把握度"，**只给建议、宿主定开口措辞**。
- **验收场景跑通**：感知"游戏到 3:30"(observed) → 用户"没睡好" → 推"可能玩太晚导致没睡好" → 带证据问 → 用户"没有，只是挂机和女友打电话" → 画像被修正。

### 额外 · 地基债：证据级引用 ✅
修了"一个事件覆盖多条原话 → 每条认知吞下全部原话当支撑、置信虚高"的老问题。consolidate 现在让 LLM 引**具体原话 id**，只挂被引的那几条。支撑去污染、置信不虚高。

### 额外 · 画像后台自动更新 ✅（阶段 4-B 起步）
聊完停手几秒（防抖）→ 后台自动 `updateProfile`（整理事件 + 重算画像 + 归因），**不挡聊天**。解决"更新画像慢"的体感（慢在后台、不挡你聊）。

### 阶段 4-B · 周期后台 ✅（正题全完成，离线绿）
- **分型衰减**：有效置信 = 原始置信 × 时间衰减因子（`2^(-age/半衰期)`），**读时算不持久化**。情绪/假设忘得快（半衰 1.5-2 天）、目标/项目中等（14 天）、趋势 7 天、特质 60 天、**事实/明确偏好不衰减**。
- **自然过期**：临时类（情绪/假设/趋势）久未印证 → 自动标失效（保留可溯源）；明确偏好/事实**永不自动失效**。
- **召回衰减门控**：淡了的、失效的认知不再被塞进回话。
- **冲突复看**：把"冲突中"的认知拿出来、并排亮正反两面证据主动问用户，回答走纠正闭环消解。
- **跨会话趋势**：反复出现的状态（"烦/累/没睡好"）规则筛频率（窗口内 ≥3 次）+ LLM 归纳 → 聚成"持续低落/高压"趋势认知（`contentType='trend'`、`formed_by='ruled'` 基于客观频率、比"特质"可信）。

## 6. ⬜ 还没做：阶段 4-A 多源感知（**要和所有者讨论方向**）

**已定的接入抽象**（地图 cell 7）：
- 所有来源（对话 / 窗口 / 未来设备）**统一抽象成"证据"**，进同一证据层。
- 每条证据必带精确时间戳；可信度绝不一视同仁（亲口 > 推断 > 观测）。
- 第一版只接对话；窗口 / 设备**留好插口**，阶段 4 再接。

**已有的最小形态**：测试台有手动"注入观察证据"口（`/api/observe`，已幂等），归因（M4）已能用 observed 证据。

**执行 AI 出过的方案草案**：MemoWeft 提供批量摄入 API `ingestObservations(subjectId, observations[], deps)`；**不在 MemoWeft 里写平台采集**（窗口标题、设备信号的采集归宿主，cell 9 边界），MemoWeft 只定义"观察怎么进来"。

**需要产品经理和所有者拍的方向**：
1. **窗口源真接到什么程度**？只定义摄入 API + 测试台手动注入，还是真接一个窗口源 demo（涉及 Electron / OS 平台 API，可能超"库"范畴）？
2. **平台采集的边界**：哪些信号采集（窗口标题 / 活跃时长 / 应用类别）？采集逻辑归宿主还是 MemoWeft 给参考实现？
3. **批量 / 时间对齐 / 节流**：设备信号量大，怎么攒批、对齐时间线、别把提炼累垮（难点 3）。
4. **隐私 / 授权**：观察类证据的授权位默认（能否上云推断）、用户怎么知情同意。
5. **多源数据质量**：观察类比对话脏，会放大现有"过度归因"的风险——要不要更保守的门控。

## 7. ⚠️ 已完成部分需要调整 / 校准的地方（**要讨论优先级**）

1. **真模型 dogfood 未系统验**：地基债（证据级引用）、后台自动更新、阶段 4-B 全套（衰减/过期/召回门控/冲突复看/趋势）**目前只过了离线单测**，真模型端到端还没系统跑。建议先 dogfood 一轮再上 4-A。
2. **慢**：实测瓶颈是**模型生成慢**（小米 MiMo 长请求 ~85 秒；MemoWeft 本地处理毫秒级、非瓶颈）。后台化已让它不挡聊天，但根治需**换快模型** 或 **给写路径单独配小快模型**（设计里写了"提炼用便宜小模型"，没落地）。
3. **参数全是草案、待运行后校准**：把握度起步分 / 衰减半衰期 / 过期阈值 / 主动询问的置信带 / 趋势窗口与频率阈值——都是拍脑袋的初值，要 dogfood 后调。
4. **闭环 conflict vs correct 由 LLM 判**：用户明确否定一条假设时，这轮 dogfood 模型判成了 `conflict`（暴露保留）而非 `correct`（划失效）。所有者当时接受"维持现状、让 LLM 判"，但这是个可能要回头看的点（要不要上"显式提问↔回答链接"强制走 correct）。
5. **召回相似度阈值门控没做**：只做了"有效置信"门控；"相似度太低的不相关认知别注入"这个老债还在（小画像时尤其会带出不相关的）。
6. **授权边界未收口**：跨宿主隔离（生活信息给不给工作助手，用 host_id）、用户查改删记忆的完整 UI——地图 cell 12 还开着。
7. **趋势 / 假设的更新而非新增**：趋势和假设目前靠衰减/过期自然淡出，没有"更新同一条"的机制，短期可能有相似多条（已加 dedup 防爆炸，但不是真正的合并）。

## 8. 关键设计决策记录（这一轮拍的板，避免重复讨论）

- 假设存 `contentType='hypothesis'`、趋势存 `contentType='trend'`（动联合类型，非表结构）。
- 主动询问**保守时机**（候选/低置信 + 把握度带内 + 一轮 1 个 + 问过不再问）。
- 纠正闭环**轻量复用**阶段 2（不上显式 respondingTo 链接）。
- 衰减**读时算不持久化**；失效语义 = 临时类超阈值标 invalidAt、稳定类不失效。
- 趋势 = **规则筛频率 + LLM 归纳**（频率客观、LLM 只命名）。
- 归因**禁 state→state**、一次只归因最近 1 现象、支撑 ≤2+1 锚点。
- 嵌入器用本地（零外传、合隐私魂）；底料严格参考 Mem0/Graphiti 自研、不拿作基座。

**还开着的开放问题**（地图 cell 12）：4-A 多源协议与时间对齐；跨宿主隔离；把握度算法的精确校准；主动询问时机的细调；成功的"可观察信号"（除主观感受外要不要别的判断依据）。
