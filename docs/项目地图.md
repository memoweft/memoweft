## 一、它是什么 + 为什么做（魂）✅

**定位**：MemoWeft 是套在大模型 / Agent **外部**的"用户认知与上下文框架"。它持续接收用户**授权的**对话与行为证据，把对一个人的理解沉淀成**独立于任何模型、可追溯、可演化、可迁移**的认知资产；并在需要用户认知的调用前，提供与当前任务相关的用户上下文、**置信度**和**边界**。

**它不做**：聊天、角色、语音、任务规划——那些是宿主的事。MemoWeft 只负责"关于这个用户，该把什么交给模型"。

**为什么做**：作者最早想做桌面助手"星瑶"，并想用自己的数据微调出"另一个我"。微调撞墙——模型刻板、数据混乱、记不住以前聊过的。由此想通：

> **对一个人的理解，不能锁死在某个模型的权重里。它必须活在模型之外——可以带走、可以持续更新。**

这就是"跨模型可迁移"成为立身之本的由来——不是技术选择，是从失败里撞出来的核心诉求。

**一句话差异**：别人的记忆框架管"存储和检索"；MemoWeft 在记忆能力之上，多做一层"**认知纪律**"（信什么、多有把握、哪些不能越界、能不能接受纠正）。

## 二、成功长什么样 ✅

- **验收是主观的**：标准是产品所有者用着的感受——"我自己用着，觉得它**真的在理解我**"，不是 benchmark 跑分。
- **"越用越懂"分两层**：
  - **事实 / 偏好层**（你说过的、做过的）→ 大概率能做到，是成熟记忆框架就能给的底料。
  - **深层假设层**（性格、没说出口的动机）→ 这是赌注，技术上没保证；靠"**低置信 + 证据校准 + 对抗确认偏误**"兜底，不追求一次猜准。
- **核心规格**："画像**可以不准、可以有偏差，但不能偏离太大**"。怎么保证不偏离太大 → 不靠算得准，靠**它知道自己可能错、并有机制去纠**：低置信假设 + 拿不准就带证据问 + 用户一纠正就改。
- **终极画面**：用很久以后，用户问"我前天睡前那个想法怎么样？"，MemoWeft 能精准找到那条、给足上下文，让宿主用了解用户的方式回答。

> 🟡 **开放**：成功的"可观察信号"还没定（除了主观感受，要不要有别的判断依据），留待测试台 dogfooding 中长出来。

## 三、和别人的异同（以 Graphiti / Zep 为镜）✅

Graphiti（Zep 的开源引擎）是目前和 MemoWeft 最像的"镜子"。

**一样的底料（抄它的思想，别当 MemoWeft 卖点）**：
- 时序知识图谱：Episode（原始数据，溯源根）= MemoWeft 的证据层。
- 事实带"有效时间窗"（valid_at / invalid_at）、双时态模型。
- 证据可溯源、保留历史不直接删（失效而非删除）。
- 三模态检索（向量 + 关键词 + 图遍历）。

**真差异（MemoWeft 多做的认知层，差异化全在这）**：
- **记 ≠ 信 / 多属性**：Graphiti 的事实只有"有效 / 失效"；MemoWeft 区分置信度 / 有效性 / 适用范围 / 相关性 / 权限，状态分候选 / 低置信 / 有限可信 / 稳定 / 冲突中。
- **来源强度分层**：用户亲口 > AI 推断 > 行为观测（Graphiti 较一视同仁）。
- **可解释假设**：推"行为可能意味着什么"（低置信、挂证据、可推翻）；Graphiti 只抽发生过 / 说过的事实。
- **冲突先暴露，不自动消解**：Graphiti 发现矛盾**自动**判定旧事实失效；MemoWeft **先进入"存在冲突"状态、不自动选边**，可带证据问用户。← **这是最根本的分水岭。**
- **禁止系统自证**：自身输出 / 用户沉默不得当证据。
- **主动带证据询问** + **授权贯穿数据生命周期**：Graphiti 不管。

**结论 / 战略**：底料（时序、有效性、溯源、召回）借 Graphiti 的**思想和数据模型**（不是代码——它是 Python + 图数据库，和 MemoWeft 的 TS + SQLite 冲突）；省下的力气全压到认知层那几条"真差异"上。

## 四、整套系统流程 / 架构 ✅

**核心结构：读写解耦 + 一份真相**（不是一条流水线——那是 v2 的错，会让人以为每说一句话都要先调模型提炼再回答，很卡）。

- **读路径（同步，服务当前对话，要轻、不阻塞）**：
  工作记忆窗口（最近 N 轮"在场"，不召回）→ 意图补全 → 回话顺带判断"该不该召回"→（需要才）召回：找相关 + 排序 → 用户上下文包（带置信度 / 适用范围 / 边界）→ 交宿主。
  多数对话窗口就够，默认**仅 1 次模型调用**；需翻旧记忆才 2 次。
- **写路径（批量，长期积累，可延后 / 可重试，不挡当前回答）**：
  原话 → 写入**证据层**（记 ≠ 信，先记下原料）→ **事件化 distill**（把近期对话总结成带情境的「事件」，挂回原话；只总结用户话，禁止自证）→ **画像 consolidate**（读事件 → LLM 提候选认知 → **MemoWeft 自算把握度** → 溯源回原话 → 更新用户模型：候选/低置信/有限/稳定/冲突中）。
  三层：**evidence 原料 → event 事件 → cognition 判断**（阶段 0/1a 已实现）。
- **一份真相（两条路径共享）**：证据层（Event · SQLite · 可查询） / 用户模型（多维认知 · 可迁移资产） / 感知缓冲（裸文件 · 定时总结）。
- **宿主边界**：MemoWeft 给理解，宿主 / 角色系统决定表达。
- **反馈分级回流（禁止系统自证）**：用户纠正 / 确认（最强）> 客观行为观测（中）> 沉默 / 点赞（仅弱反馈）> 助手自身输出（**永不**当证据）。
- **周期性后台（离线）**：Profile 沉淀 · 按认知类型分别衰减 · 冲突暴露不急消解 · 主动带证据询问。

> 那个"昨晚没睡好"的例子，就是从这张图走了一趟：写路径早把"游戏开到 3:30"写进证据层 → 你开口走读路径 → 宿主表达成带证据的提问 → 你纠正后从反馈回流回到写路径 → 修正用户模型。

## 六、数据模型 🟡（草稿 · 借 Graphiti + MemoWeft 认知层）

> ⚠️ 这是表结构层面，属"必须人拍板"的红线（旧 D-018 精神）。以下是**草稿基线**，字段都可改，下方列了未定的分叉。

```
证据层 evidence  (≈ Graphiti Episode；阶段 0 已实现 src/evidence/)
  id                   主键
  subject_id           哪个用户
  source_kind          spoken亲口 / inferred推测 / observed观察（来源强度）
  host_id              来自哪个宿主
  origin_id            原始消息号·幂等防重写（可空·唯一索引）
  occurred_at          事情发生时间（双时态·借 Graphiti）
  recorded_at          MemoWeft 存入时间
  raw_content          原文
  summary              召回载体；v1 = raw_content，阶段1 再 LLM 抽（替代 topic）
  allow_local_read     授权：能否给本地 Agent
  allow_cloud_read     授权：能否发云端（默认跟随 privacyMode 配置）
  allow_inference      授权：能否据此推画像/动机
  corrects_evidence_id 纠正指向（可空·逻辑阶段3 M6）
  -- topic 作为"相关性机制"已废（太粗）；找相关靠 summary（阶段1 向量精挑）。

事件 event  (情境化层·在原料与判断之间；已实现)
  id, subject_id, summary(带情境总结), occurred_at(锚到最早), created_at
event_evidence  (事件覆盖了哪些原话证据)
  event_id, evidence_id
  -- distill 把"未整理"的近期对话总结成事件；consolidate 读事件、溯源回原话。

认知条目 cognition  (判断层·多维用户模型；阶段 1a 实现)
  id, subject_id, content(一句话认知)
  content_type   fact事实/preference偏好/goal目标/project项目/state状态/trait特质
  formed_by      stated亲口/observed观察/ruled规则/inferred LLM推测   ← 来源强度
  confidence     置信度 0~1000（MemoWeft 自算，非 LLM 自报）
  cred_status    candidate候选/low低置信/limited有限/stable稳定/conflicted冲突中
  scope          适用场景（可空=通用）
  valid_at, invalid_at   有效时间窗（借 Graphiti，可空）
  created_at, updated_at
  -- 授权位不在此层：授权挂在 evidence（原料层）；判断层只存判断（记≠改画像）。

溯源链 cognition_evidence  (谁支撑/反对这条认知)
  cognition_id, evidence_id, relation(support支持/contradict反对)
```

**已定（阶段 1a 确认）**：cognition 一张表 + cognition_evidence 溯源链（不拆实体/边两张）；授权位归 evidence 不进 cognition；6 维属性如上。
**剩余分叉**：`valid_at/invalid_at` 粒度与 `confidence` 的关系（运行后校准）；confidence 的具体算法（见开放问题"把握度怎么算"）。

## 七、多源感知 🟡（接入抽象已定，关联机制定了草稿）

**接入抽象（已认可）**：
- 所有来源（对话 / 窗口 / 未来设备）**统一抽象成"证据"**，进同一证据层。
- 每条证据**必带精确时间戳**；可信度**绝不一视同仁**（亲口 > 推断 > 观测）。
- 第一版只接**对话**；窗口 / 设备留好插口，阶段 4 再接。

**时间对齐 / 跨源关联（归因 M4 的前提，已定草稿）**：
- **粗筛 = 时间窗**：按时间范围 SQL 拉候选（便宜，天然贴合"昨晚 / 前天"这类归因查询）。
- **相关性 = 向量语义召回**（已定，抄 Graphiti）：summary 作为被嵌入的文本，用语义相似度找相关，区分"看着同类、其实是不同项目 / 不同事"。**不用 topic**（太粗，已废）。可叠加关键词（FTS5）与关联表遍历，呼应 Graphiti 三模态。
- **召回边界（V1 契约，2026-07 定）**：一个 subject 对应一个 Retriever 实例（独立向量库文件，如 `memoweft-vectors.<subjectId>.db`）；`Conversation` 注入点另有 subjectId 硬过滤兜底（`src/pipeline/conversation.ts`）。**非死规则，升级前提**：出现"单进程需服务多 subject 的宿主"时，升级为 vectors 表加 subject 列、Retriever 接口带 subjectId（B 方案）。
- 剩下子问题：嵌入模型本地 vs 云端（见难点 2 / 开放问题）。

**底料策略（已定）：严格参考、自研、接口隔离。**
- **严格参考 Mem0 / Graphiti 的方法和逻辑**（读其开源 + 论文）：事实抽取的 prompt 与粒度、更新 / 去重策略、混合检索（向量 + 关键词 + 时间）的排序、多级记忆划分。
- **但自研成 MemoWeft 自己的最小召回层**（几百行，不背依赖）——把它们的智慧抄成代码，关键处改成 MemoWeft 纪律（如"更新 / 去重"改成**冲突先暴露**而非自动覆盖）。
- **不拿 Mem0 作基座**：单人 / 几千条规模用不上其重型混合检索；且会破"最小依赖"、锁定、跟其自动管线打架。
- **接口隔离（保险丝）**：`存储 / 召回` 放在一个可替换接口后面，默认自研实现；将来若规模真上来，Mem0 可作另一实现 slot 进去，不锁死。证据真相与认知层**永远在 MemoWeft 自己手里**。

> 例："昨晚没睡好" → 时间窗拉取昨晚全部证据 → summary 精挑出真正相关的几条（区分"挂机"和"真玩"得靠后续用户纠正，见认知层规则）。

## 八、认知层规则 🟡（MemoWeft 的魂 · 草稿待逐条确认）

> 这是 MemoWeft 区别于普通记忆框架的全部价值所在。以下每条都是**草稿、可改**，已尽量用大白话。

1. **记下来 ≠ 立刻拿来改画像**：把你的话、行为先存进证据库，是免费的、随便记；但**要不要据此给你下结论、更新画像**，是另一道门槛。脏东西被记没关系，关键是别让它轻易变成"对你的判断"。
2. **来源不一样，可信度不一样**：同一件事，你**亲口说**的，比 AI **猜**的、比后台**观察**到的可信得多。三种来源绝不能一样对待。
3. **每条理解是多面的，不是一个标签 / 一个分数**：一条认知同时带"这是什么类型、哪来的、什么时候、什么场景、有多确定、能不能外传"等多个侧面（见 cell 6）。
4. **它不能拿自己的话当你的证据**：MemoWeft 自己说过的、你没吭声 / 随手点了个赞，**都不算"用户证据"**。只有你主动说的、确认的、纠正的，才算数。（防止它自说自话、越证明越信自己。）
5. **发现矛盾，先标"有冲突"，不急着判谁对**：新旧对不上时先挂起来，分别查——可能你真变了，也可能换了场景、一时情绪、说反话、或它记错了。必要时降低把握、缩小适用范围、两条都留着、等更多证据，或**带证据问你**。←（这正是 Graphiti"自动作废旧事实"做不到的，是 MemoWeft 的分水岭。）
   - **阶段 2 已实现**：consolidate 增量区分 **correct**（用户【明确纠正】→ 旧标失效保留 + 新采纳）与 **conflict**（模糊矛盾 → 标 `conflicted` 暴露，不替换）。带证据主动询问留阶段 3（M5）。
6. **它会推测"你行为背后的意思"，但只敢低声说**：比如"可能玩游戏太晚导致没睡好"——这种推测一开始把握很低、挂着证据、随时能被你推翻。这是 MemoWeft 最想验证的能力。
7. **它得防着自己"越用越自以为懂你"**：挑上下文时故意给"反对当前判断的证据"留位置；推测的把握度不能只涨不跌；没把握时要老实告诉宿主"这条我也不太确定"。
8. **不同的东西，过期速度不一样**：此刻的情绪很快就该忘；进行中的项目看它结没结束；你明确说过的偏好，哪怕很久没提也不该自动失效。**不能一刀切地"越久越不信"。** ←（v1 已实现：临时类 `state` 置信封顶、永不"稳定"，见 `config.consolidation.transientTypes`；完整版按真实时间衰减留后续。）
9. **它只说"我收下到什么程度"，不说"我证明这是真的"**：一条理解会被标成候选 / 低置信 / 有限可信 / 稳定 / 冲突中，但 MemoWeft 不会宣称"我证明了这是真的"。而且**模型自己报的"我有 80% 把握"不能直接当数**，MemoWeft 另有一套判断把握度的依据。
10. **你能随时看、改、删自己的记忆**：MemoWeft 存了什么、对你下了什么判断，你都可以**直观地查看**；不对的可以**改**、不想要的可以**删**。你对自己的记忆有完全的控制权，它不是一个黑箱。

> 🟡 开放：原先那条更抽象的"授权边界"（能不能传云端 / 能不能拿去推测心理 / 生活信息能不能给工作助手）暂未列为规则，先记在开放问题里，要不要单独成条以后再说。

## 九、宿主边界 ✅（MemoWeft 给理解，宿主定表达）

**MemoWeft 只管（理解层）**：提供对用户的理解、与当前请求相关的事实与历史、把握度、适用范围、可能的敏感点、哪些不能假设、哪些存在冲突、用户的沟通偏好。

**宿主 / 角色系统管（表达层）**：最终语气、角色口吻、是否主动补充、是否执行操作、怎么组织回答、产品与安全策略。

**为什么这条线这么切（关键例子）**：MemoWeft 是**插件**——
- 装在 **Hermes** 上 → 它就成了 Hermes 助手，用户用 Hermes 去个性化角色；
- 装在 **星瑶** 上 → 它就是星瑶的脸。
- **同一份"对你的理解"，配不同宿主的不同人格。** MemoWeft 不该自带语气 / 角色，否则就锁死在一个宿主里，违背"可迁移"（cell 1）和"框架/库"形态。

## 十、已知难点 / 风险 ✅（横向对照 Mem0 / Zep / Graphiti 等后确认的真坑）

1. **（最严重）用规则抽"性格 / 特质"做不到**：研究表明提示法预测性格基本无效。**对策**：分两类——行为类（频率、时间模式）用规则算，准，是地基；特质 / 假设类借 LLM 悟，产物当**低置信假设**，靠证据校准。
2. **嵌入模型：可替换；当前用本地 Ollama `bge-m3`**（1024维，多语言/中文）。原定"云端优先"，但用户本地有 bge-m3 → 直接用本地（零外传、零成本，更合隐私魂）。换云端只改 `.env` 的 `DLA_EMBED_*`。**难点 2 初步验证**：真实数据上 bge-m3 中文召回，"喝什么"准确挑出"喜欢喝茶"；分项目/生活的更难场景仍待更多 dogfood。
3. **别让本地机器被频繁"提炼"累垮**：攒批处理、提炼用便宜小模型（= 写路径解耦）。
4. **旧信息会"压住"新信息**：过时但措辞刚好对上的旧信息会排在更新信息前面。**对策**：降把握度不能只是数字变小，要小到挑上下文时根本轮不到它。
5. **聊得越久越冲淡关键偏好**：研究表明 AI 对偏好的遵守率随对话变长大幅下降。**对策**：关键偏好存用户模型、每轮调用前重新注入。
6. **11-A 债 · 写路径膨胀（2026-07 登记）**：consolidate 每轮全量注入 active 画像，prompt 随画像线性涨。**修复方向** = 相关性限定注入（retriever 对新事件文本检索 top-K + 全部 conflicted + 最近 N 条），**且必须配防重网**（new 候选落库前对 active 认知做相似度查重，高相似降级 reinforce 或丢弃）——代价：限定注入漏召回会致重复创建，故防重网先行、可独立落地。**触发条件**：dogfood 观察写路径落盘的 profileSize/promptChars 曲线（已接线落盘），到疼点由人拍板，非现在修。

## 十一、技术栈 / 工程形态 🟡（沿用旧决策草稿，可改）

- **语言：TypeScript**。与宿主同栈（Hermes = Electron/TS，星瑶 = TS），零集成成本；类型系统把字段约束焊在编译期。
- **存储：SQLite**（Node 内置 `node:sqlite`，零原生编译、零外部依赖；旧项目因 `better-sqlite3` 装不上才换的，沿用）。
- **向量：已定要上**（抄 Graphiti 的语义嵌入召回）。SQLite 上可用向量扩展（如 `sqlite-vec`）+ FTS5 关键词，仍属轻量。
- **嵌入器：云端优先，可替换**。嵌入模型做成可插拔的一处部件，默认云端，将来可换本地——**不写死成全云端或全本地**。
- **模型：** 官方 TS SDK / OpenAI 兼容接口（换模型只动一处）。
- **依赖取向：能参考借鉴就参考借鉴，不盲目造轮子；但核心自有、重依赖慎入。**
  - 自由 **参考 / 借鉴逻辑与模式**（Mem0 / Graphiti 的方法、成熟算法、设计套路）——优先，不重造。
  - **小而专、可替换的库**：明显省事又不绑死的，可以用（别什么工具函数都手写）。
  - **拒绝**：重框架（LangChain）、重基座（Mem0 作地基）、会用自身哲学覆盖 MemoWeft 理解机制的东西。
  - **永远自有、不外包不锁定**：证据真相 + 认知层（MemoWeft 的差异化）。
- **形态：框架 / 库**（npm 包），被宿主 `import`，随宿主进程生死，无独立服务。
- **运行 / 构建（已搭好）**：dev 用 Node v24 **原生 TS**（strip-only，直接跑 `.ts`）；发布用 `npm run build`（`tsconfig.build.json` + TS 5.7 `rewriteRelativeImportExtensions` 把 `.ts` 导入重写为 `.js`、出 `dist/`）。
  - ⚠️ **原生 strip-only 模式禁用"需代码生成"的 TS 语法**：**参数属性**（`constructor(private x)`）、`enum`、`namespace` 等都不能用——用显式字段 / 普通常量替代。
  - 脚本齐备：`typecheck` / `test`（仅扫 `tests/`，不碰 `reference/`）/ `testbench` / `build`。
