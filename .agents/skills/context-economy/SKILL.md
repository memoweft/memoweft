# SKILL · context-economy（开工省 token：只加载该加载的）

**何时用**：每次开工 / 接手 / 换任务时，决定"读什么、不读什么"。

**目的**：让任何 AI agent 工作时只加载极少上下文，不烧 token 重读、重推、通读。

## 开工读取顺序（够用即停）

1. **`STATE.md`**（白板·此刻状态 + 可用接口 + 下一步）——**永远先读，极短**。多数情况读完它就能开工。
2. **`AGENTS.md`**（契约 + 工作流）——心里有数即可，不必反复读。
3. 任务点名的 **`docs/项目地图.md` 单个 cell**——用 Grep 按章节名（如 `## 七、`）或关键词定位，**只读那一格，绝不通读 600 行地图**。
4. 要改的 **代码文件**——优先 Grep 定位 + 用 `STATE.md` 已列的接口签名；需要才整段读（用 offset/limit，不整文件灌）。

## 不要做（烧 token 的坏习惯）

- ❌ 通读整份 `项目地图.md` / 整个 `src/` 树 / 整个 `reference/`。
- ❌ 读 `LOG.md`（磁带）——只在追溯"当初为何"时翻。
- ❌ 重读**已在当前上下文**里的文件 / 刚 Write/Edit 过的文件（改动若失败工具会报错）。
- ❌ 用整文件 `Read` 找一个符号——改用 `Grep`。
- ❌ 让方案 / 讨论把不相关的 cell、不相关的旧代码全搬进上下文。

## 完工写回（保持白板便宜）

- **改写 `STATE.md`**（反映此刻：阶段、可用接口、断点、下一步；≤40 行，别追加流水账）。
- **追加 `LOG.md`**（一段：日期 + 做了什么 + 定了什么决策 + 验证）。
- 仅当**决策变了**才改 `项目地图.md` 对应 cell。

## 任务间靠"接口"交接，不靠"过程"

下一个任务只需 `STATE.md` 的「当前可用接口」，**不必读上一个任务的实现与讨论**。设计在地图、状态在白板、历史在磁带——各取所需，互不灌入。
