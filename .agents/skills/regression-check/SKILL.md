# SKILL · regression-check（跑测试台 + 护栏 + 落盘）

**何时用**：自查通过后，交用户 dogfood 之前 / 之后。

**目的**：确认没回归、内幕已落盘可供诊断、废弃物没复活。

## 1. 自动护栏（底线，不是主角）

- 跑结构性不变量测试（不调模型、快、确定）：写入→读出一致、可复现（同输入同输出）。
- **防回潮断言**：表结构里不得出现已废字段 / 机制（如把 `topic` 当相关性主键、单一 `weight` 字段、State/Profile 双层时效残留）。
- Event / 认知存储层**不得有物理删除以外的越权接口**；删除须是用户可控的"查看/修改/删除"（地图 cell 8 第 10 条），不是系统偷偷删。

## 2. 测试台落盘（关键 —— 让执行 AI 能直接读）

- 每轮对话把**全部内幕**追加进 `logs/run-*.jsonl`：用户输入、召回了什么、生成的假设 + 置信度、可信状态、有没有冲突、是否主动询问、模型调用次数、画像变更、报错。
- 数据库 `dla.db` 保持可被 SQLite 直查。
- **新增任何透视字段，必须同时写进落盘日志**，否则诊断时看不到。

## 3. 用户 dogfood 后

- 执行 AI **直接读 `logs/run-*.jsonl` + 查 `dla.db`** 定位问题，不靠用户复述。
- 把发现的偏差带回 task-planning（要改方向 / 决策）或 safe-implementation（纯实现 bug）。

> 验收标准是主观的（用户"觉得被懂了"）；自动护栏只保证"没倒退、没回潮"，不替代用户判断。
