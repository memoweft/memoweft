# SKILL · task-planning（开工前出方案）

**何时用**：开始任何任务 / 阶段、动手写码之前。

**目的**：把"要做什么、碰到哪些已定决策、有什么不确定"想清楚，先交人确认，绝不闷头开写。

## 步骤

1. **读上下文（省 token，见 context-economy）**：先读 `STATE.md`（白板：当前阶段 + 可用接口 + 下一步）。再用 **Grep 按需定位** `docs/项目地图.md` 的相关 cell（如 cell 8 认知规则 / cell 6 数据模型 / cell 12 开放问题），**只读用到的那格，不通读 600 行**。
2. **只摘相关部分**：本任务涉及哪几格、哪几条规则、哪几个开放问题。不相关的不堆。
3. **出方案**：函数 / 模块签名级别说清打算怎么做；涉及数据结构的，先描述表 / 字段。
4. **列影响面清单**：本改动会碰到哪些文件 / 接口 / 已落库决策；会不会动到核心机制或表结构。
5. **标出不确定点**：哪里有多种做法都行、取决于用户；哪里撞上 cell 12 的开放问题。
6. **碰决策必停**：若方案需要改某条已定决策、改 Event/认知表结构、或地图没覆盖 → **停下来问人**，不自行决定。

## 产出（交人确认后才进 safe-implementation）

- 方案（做法 + 签名）
- 影响面清单
- 不确定点 / 需用户拍板的问题

> 红线：未经确认不得进入实现。"先想清楚再动手"是本项目反复强调的纪律。
