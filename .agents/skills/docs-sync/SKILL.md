# SKILL · docs-sync（完工后同步文档）

**何时用**：一个任务 / 阶段验收后。

**目的**：保持**文档 = 代码**，且让下次开工省 token。

## 先更新这两个（便宜、每次都做）

1. **改写 `STATE.md`**（白板）：当前阶段、`当前可用接口`（新增/改动的接口签名挪进来）、进行中断点、下一步。保持 ≤40 行，**改写不追加**。
2. **追加 `LOG.md`**（磁带）：一段 = 日期 + 做了什么 + 定了什么决策 + 验证结果。只增不改。

> 任务间靠 `STATE.md` 的接口签名交接，下一个任务就不必读你的实现——这是省 token 的关键。

## 决策变了，才改地图（按 cell，别通读）

1. **状态总览表**：把相关格子的状态改对（🔴→🟡→✅）。
2. **cell 5 分阶段表**："产出 / 验收"列勾掉已完成的阶段，标出当前在哪一阶段。
3. **cell 13 现有代码**：更新"可复用 / 已废 / 处理方式"。
4. **cell 12 开放问题**：撞到并定了的，划掉并标"已定 + 结论"；新冒出的，追加。
5. **决策落地**：本次若**经用户拍板**定了新方向 / 改了旧决策 → 写进相关格子，并在 cell 8 或对应格附一句"何时、为何这么定"。

## 纪律

- **只搬"经用户确认"的东西**进地图；AI 没权自己往地图里加决策。
- 地图每格仍是**草稿、可改**——同步是反映现状，不是把它焊死。
- 改动要可追溯：保留"曾经怎么想、为何改"的痕迹（哪怕一句话），方便日后回头打开重定。
- 对外 README（含 zh-CN）与 STATE.md 状态一致，README 不得含手工维护的测试数字（测试数走 CI badge）。
- 旧工程 `../DLA_project` 不动；它永远是"可参考"。

## 产出

一句话告诉用户：这次更新了地图的哪几格、哪条决策落地了、还剩哪些开放问题。
